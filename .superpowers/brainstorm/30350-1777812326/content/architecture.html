ccmux — Architecture

Three processes, clean separation of concerns

① ccmux sidebar
Ratatui TUI running in a tmux split pane.

• Lists all Claude panes, session-wide, grouped by window
• Hybrid layout: compact rows, selected expands
• Worktree / new window flows
• Spawns notify-worker on first open
• Refreshes every 500ms
② ccmux notify-worker
Tiny background process, daemonised by sidebar.

• Polls tmux every 2s
• Fires macOS notification on WaitingInput
• Sets @ccmux_alert on the tmux window
• Clears alert when user focuses pane
• Self-exits when session ends
③ ccmux status
Fast one-shot subcommand for tmux status-right.

• Reads pane content, prints ●/◐/○/?
• Reads @ccmux_alert → flashes ⚠ when alert set
• Called as #(ccmux status --window #{window_id})
• Zero startup cost
Toggle flow (prefix+Tab)
tmux.conf: bind Tab run-shell "ccmux toggle"
└→ check @ccmux_sidebar_<session>_<window> variable
└→ if alive: tmux kill-pane (hide)
└→ if not: tmux split-window -hb -l 50 "ccmux sidebar" (show)
└→ store new pane ID in @ccmux_sidebar_<session>_<window>
State storage — all in tmux variables, no files needed
@ccmux_sidebar_<session>_<window> pane ID of sidebar (per window)
@ccmux_notify_pid PID of running notify-worker
@ccmux_alert_<window_id> 1 = this window has a waiting session
@ccmux_color colour135 etc (per window, set at creation)
Multi-server support
Config lists servers to monitor. Sidebar aggregates panes from all of them.
Each tmux command fan-out: default server + any configured extras (-L name).
Server label shown in window group headers when multiple servers active.
